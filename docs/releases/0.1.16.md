# HQBase Community 0.1.16

- Community-to-Pro upgrades keep all newly generated Pro secrets isolated to the candidate Worker
  version instead of creating an intermediate production Community version.
- Existing Community bindings and secrets are inherited from the exact active version recorded
  during target verification, so backup and migration checkpoints cannot drift silently.
- Failed upgrade steps can be retried from the persisted checkpoint without reloading the page.
- Candidate verification tolerates normal Cloudflare version-propagation delay while Community
  continues serving production traffic unchanged.
